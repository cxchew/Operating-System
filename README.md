# SECR2043 Operating Systems: Course Portfolio 🐧💻

Welcome to my core repository for **SECR2043 Operating Systems** at Universiti Teknologi Malaysia (UTM). This portfolio houses a structured log of practical laboratory experiments, low-level shell engineering, system monitoring tasks, and a multi-threaded synchronization engine.

---

## 📂 Laboratory Core Index

### ⚙️ Lab Assessment 1: Command-Line Essentials & Native C Integration
* **Lab Summary:** Focuses on baseline environment navigation, file-stream operations, and compiler toolchain basics. 
  * *Activity 1:* Explored Linux filesystem command sets including absolute path routing (`pwd`), mapping hidden initialization scripts (`ls -a ~`), workspace deployment (`mkdir`), and text streaming (`echo > hello.txt`).
  * *Activity 2:* Used the terminal text editor `nano` to draft native C logic, managed binary compilations using the GNU Compiler Collection (`gcc`), and integrated positional command-line flags (`argc`, `argv[]`) to engineer dynamic parameter handling tools.

### 🧵 Lab Assessment 2: Process Monitoring, Hierarchy & Scheduling
* **Lab Summary:** Investigates the active process lifecycle, multitasking behavior, and dynamic CPU scheduling interventions.
  * *Prerequisites:* Configured background processing tasks by fetching, compiling, and spinning up continuous asynchronous mock scripts (`./mainprocess &`).
  * *Process Management:* Inspected active processes using target filtering flags (`ps -eo`), sorted systems by descending RAM use (`--sort=-pmem`), traced process trees using parent-child mapping configurations (`pstree -p -t`), altered scheduling limits using root commands (`renice`), and executed terminal termination sequences (`killall`).

### 🛡️ Lab Assessment 3: Bash Shell Scripting & POSIX Permissions
* **Lab Summary:** Combines scripting automation with system asset security hardening techniques.
  * *Activity 1:* Evaluated complex inline script text matrices to track tracking and directory execution behaviors, drafting parent-to-child folder trees from the root environment.
  * *Activity 2:* Engineered a specialized Bash scripting loop (`myname2a.sh`) to automatically deploy directory systems from files while cleaning delimiter tokens. Tested access configurations (`drwxrwxr-x`) and designed a security remediation script (`myname2c.sh`) utilizing octal and symbolic modes (`chmod`) to isolate Owner, Group, and Public access profiles.

## 🍝 Featured Project: Restaurant Table Management System

An advanced multi-threaded simulation engine designed to programmatically resolve the classical **Concurrency Synchronization Problem** inside unmanaged shared memory registers.

* **Synchronization Mechanics:** Combines counting semaphores (`threading.Semaphore`) to control finite dining space pools with binary mutex components (`threading.Lock`) to eliminate race conditions on shared income and table allocation data structures.
* **Thread Telemetry Display:** Integrates a visual monitoring dashboard charting threads in real-time as background tasks shift from blocked waiting states to active data execution contexts.
