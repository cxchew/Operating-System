# Lab Assessment 1: Linux Command-Line Essentials & Native C Integration 🐧⚙️

## 1. Lab Summary
This laboratory establishes baseline competency in operating system interaction by bridging terminal configuration environments with low-level compilation pipelines. The assessment is divided into two operational segments:

* **Activity 1: Basic Linux Commands** Investigated core filesystem orchestration and text streaming. This involved absolute path resolution (`pwd`), mapping hidden initialization dotfiles (`ls -a ~`), synthesizing dedicated workspaces (`mkdir ~/os_lab`), and demonstrating data persistence by routing terminal input streams into static storage channels (`echo > hello.txt`).
* **Activity 2: C Programming in Linux Environment** Shifted from shell management to system asset creation. Utilized the inline `nano` text editor to draft native C programs. Executed compilation commands via the GNU Compiler Collection (`gcc`) to build running binaries, and implemented positional command-line argument bindings (`argc` and `argv[]`) to engineer dynamic calculation and stream mirroring tools.

---

## 2. Technical Reflection

### What I Learned
* **Decoupling from Graphical Shells:** Interacting directly with the underlying Linux kernel filesystem via raw terminal vectors built deep confidence in standard shell configurations. I learned how path resolution structures work behind the scenes.
* **The Compilation Pipeline:** Manually managing the `gcc` execution loop highlighted the transformation steps that translate high-level readable source code into low-level operational machine assets.
* **Dynamic Argument Binding:** Implementing `argc` and `argv` structures in C showed me how parameters travel safely from user shells across standard memory lines straight into independent running routines.
